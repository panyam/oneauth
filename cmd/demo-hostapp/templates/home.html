What is this?

This is {{.App}}, a demo app that registered itself with the central Auth Server on startup. It has its own user database (separate from the Auth Server and other apps). After you log in, it can mint resource-scoped JWTs for you — tokens that any Resource Server sharing the same KeyStore will accept.

Step 1: Get a Resource Token

Calls MintResourceToken() — signs a JWT with this app's secret, embedding your user ID, scopes, and quotas as claims.

Step 2: Validate Against a Resource Server

  

    Send the token to a Resource Server's POST /validate endpoint.
    The Resource Server reads the kid from the JWT header,
    looks up the matching signing key, and verifies the signature.
    It has never communicated with this app — it trusts the shared KeyStore.
  

  

    
    
  

  

    Try both http://localhost:4001 (Resource Server A) and
    http://localhost:4002 (Resource Server B) — the same token works on both.
